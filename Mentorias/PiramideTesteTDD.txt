Principais testes de um dia a dia de um DEV

Muito importante* Teste Unitário - Lida com uma pequena parte do código com pequenos testes / realizado na menor parte testavel de uma aplicação / simples pequenos e rápidos
Teste de integração - Teste com junção de mais de uma parte de código / testa um conjunto de unidades interagindo entre si (interfaces, APIs, Banco de Dados) / por isso se torna mais complexo do que um teste unitário
Teste operacional - Analisa se o sistema está operacional / Simula o comportamento do usuário final em nossa aplicação / utiliza muito o BDD / Testes mais complexos com uma escolha mais criteriosa
Teste de Regressão - Analisa um pedaço do código com um bug especifico garante que o que já funciona não deixe de funcionar apesar das novas atualizações
Teste Funcional - Valida cada uma das funcionalidades está construido conforme o esperado
Teste de interface - 
Teste de configuração e segurança - Teste que analisa as configurações para que o software funciona em diferentes celulares ou navegadores de forma segura
Teste de performace - Teste que determina o quanto de acessos o software aguenta, testa situações imprevistas
Teste de Carga - Quantas pessoas acessando o software o site aguenta
Teste de Stress - 
Teste de instalação/Desinstalação - Teste para verificar se a instalação e desinstalação estão operacionais

TDD (Test Driven Development)

Desenvolvimento orientado a testes

Pensa na qualidade do software

BDD (Desenvolvimento orientado a comportamento - Complemento ao TDD)
é uma técnica de desenvolvimento Ágil que encoraja colaboração entre desenvolvedores, setores de qualidade e pessoas não-técnicas
Pensa na funcionalidade

Fluxo de teste do desenvolvimento: Dado, Quando, Então

Pontos positivos:
No final o cliente fica satisfeito. Está dentro da espectativa do cliente
melhor comunicação do time de desenvolvimento
Documentação mais dinâmica

Pirâmide de testes
Base - Testes de Unidades - feitos em maiores quantidades
Meio - Teste de integração - feito em menos quantidade que o de unidades
Topo - Teste de sistemas/ponta a ponta/end-to-end - feito em menos quantidade que o de integração
